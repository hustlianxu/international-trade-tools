## 1. 软件简介

外贸助手是一款基于微信数据解析的 AI 辅助工具，帮助外贸业务员：

- **自动解析微信聊天记录**（文字、语音、图片），准实时（可开关）
- **西语+普通话语音转文字**：支持西班牙、墨西哥、阿根廷等各国口音
- **AI 分析客户需求**：自动提取询价、报价、订单、物流等需求
- **待办事项管理**：汇总已办/待办，按天提醒，超期标红

### 工作流程

```
微信收到消息 → 自动解析 → 语音转文字 → DeepSeek分析 → 生成待办 → 定时提醒
```

## 3. 快速开始（5分钟）

### 步骤 1：获取源码

```bash
git clone https://github.com/hustlianxu/international-trade-tools.git
cd international-trade-tools
```

### 步骤 2：一键打包为桌面应用

**macOS**（在 Mac 上执行）：
```bash
bash build_mac.sh
```
完成后在 `dist/` 目录生成 `外贸助手.app`，双击即可运行。

**Windows**（在 Windows 上执行）：
```
双击 build_windows.bat
```
完成后在 `dist\TradeTools\` 目录生成 `TradeTools.exe`，双击即可运行。

> 首次打包需 5-10 分钟（下载依赖），后续打包很快。

### 步骤 3：获取 API Key

软件需要两个 API Key：

#### 3.1 DeepSeek API Key（必须，月费<1元）

1. 访问 https://platform.deepseek.com/
2. 注册并登录
3. 点击「API Keys」→「创建 API Key」
4. 复制 Key（格式：`sk-xxxxxxxx`）
5. 充值 10 元即可用数月

#### 3.2 语音识别 API Key（按引擎选一个）

| 引擎 | 注册地址 | 费用 | 推荐场景 |
|---|---|---|---|
| **火山豆包**（推荐 Win） | https://console.volcengine.com/speech | ~7元/月 | Windows 用户 |
| **MLX Whisper**（推荐 Mac） | 无需 Key | 0元 | Mac M3 用户 |
| **OpenAI** | https://platform.openai.com/ | ~11元/月 | 最高准确率 |

**火山豆包获取方式**：
1. 注册火山引擎账号
2. 开通「语音技术」→「录音文件识别」
3. 在控制台获取 App ID 和 Access Token

### 步骤 4：配置并运行

1. 双击打开 `外贸助手.app`（Mac）或 `TradeTools.exe`（Windows）
2. 进入「**配置**」页
3. 填入：
   - 微信 db_storage 路径（见下方说明）
   - DeepSeek API Key
   - ASR 引擎对应的 Key
4. 点击「**💾 保存配置**」
5. 切换到「**准实时监听**」页，点击「**▶ 启动监听**」

### 微信 db_storage 路径查找

**Windows**：
```
C:\Users\<你的用户名>\Documents\xwechat_files\<wxid_ xxx>\db_storage\
```
方法：打开「文件资源管理器」→「文档」→「xwechat_files」→ 找到你的 wxid 文件夹 →「db_storage」

**macOS**：
```
~/Library/Containers/com.tencent.xinWeChat/Data/Library/Application Support/com.tencent.xinWeChat/2.0b4.0.9/<wxid>/db_storage/
```
方法：Finder → 菜单「前往」→「前往文件夹」→ 粘贴上面路径（替换 wxid）

> 如果找不到，在微信中随便发一条消息，然后用系统搜索 `db_storage` 文件夹。

## 4. 详细配置指南

### 4.1 配置文件位置

配置文件保存在用户目录（非程序目录）：

| 系统 | 路径 |
|---|---|
| Windows | `%APPDATA%\trade-tools\config.yaml` |
| macOS | `~/Library/Application Support/trade-tools/config.yaml` |

你也可以在软件的「配置」页直接编辑，点击保存即可。

### 4.2 ASR 引擎选择

| 你的设备 | 推荐引擎 | 原因 |
|---|---|---|
| Mac M1/M2/M3 | `mlx_whisper` | 本地运行，免费，隐私好，7秒/分钟 |
| Windows 无独显 | `volcengine` | 云端，不挑硬件，~7元/月 |
| 追求最高准确率 | `openai` | 西语 WER 2.7%，~11元/月 |

### 4.3 手动编辑配置文件

如需高级配置，可直接编辑 `config.yaml`：

```yaml
# 微信配置
wechat:
  db_storage_path: "C:/Users/xxx/Documents/xwechat_files/wxid_xxx/db_storage"
  process_name: "WeChat.exe"        # Win: WeChat.exe / Mac: 微信
  realtime_enabled: true
  poll_interval_ms: 100              # 轮询间隔，越小越实时
  debounce_ms: 200
  watch_talkers: []                  # 留空=监听全部，或指定 wxid 列表
  ignore_talkers:                    # 忽略的会话
    - "filehelper"
    - "weixin"

# ASR 引擎
asr:
  engine: "volcengine"               # volcengine / mlx_whisper / openai
  volcengine:
    app_id: "你的火山AppID"
    access_token: "你的火山Token"
    model: "bigmodel"
  mlx_whisper:
    model: "mlx-community/whisper-medium-mlx-8bit"
    language: ""                     # 留空=自动检测中西语
    initial_prompt: "外贸业务，询价，报价，订单，发货，物流，提单，信用证，LC，FOB，CIF"

# DeepSeek
llm:
  deepseek:
    api_key: "sk-xxxxxxxx"
    model: "deepseek-chat"
    temperature: 0.3

# TODO 提醒
reminder:
  granularity: "daily"               # daily / hourly
  daily_time: "09:00"
  overdue_days: 3

# 存储
storage:
  db_path: ""                        # 留空=自动用用户目录
```

## 5. 功能使用

### 5.1 准实时监听

1. 确保**微信已登录并运行**（软件需要从微信进程内存读取解密密钥）
2. 在微信中**随便打开几个聊天窗口**（确保数据库已加载到内存）
3. 切换到「准实时监听」页
4. 点击「**▶ 启动监听**」
5. 状态变为「监听中...」（绿色）
6. 当客户发来新消息时，日志区会实时显示：
   - 收到消息的通知
   - 语音转写进度
   - DeepSeek 分析结果摘要
   - 新增的待办事项

**停止监听**：点击「⏹ 停止监听」

> **开关控制**：监听可随时启停，不监听时不会消耗任何资源。

### 5.2 手动语音转写

如果只想转写某条语音（不启动监听）：

1. 在微信中找到语音文件（.silk 格式）
   - Windows: `xwechat_files/<wxid>/msg/` 下的 .silk 文件
   - 或导出语音消息
2. 切换到「语音转写」页
3. 点击「选择文件...」选择 .silk 文件
4. 点击「🎙 开始转写」
5. 等待几秒，结果显示在下方文本框

> Mac M3 约 7 秒/分钟音频；火山豆包约 2-5 秒返回。

### 5.3 待办事项管理

1. 切换到「待办事项」页
2. 点击「🔄 刷新列表」查看所有待办
3. 点击「📋 生成提醒」生成今日提醒文案
4. 选中一条待办，点击「✓ 标记完成」可标记为已办

**待办分类**：

| 分类 | 说明 | 西语对应 |
|---|---|---|
| 询价 | 客户问价格 | cotización, precios |
| 报价 | 需要报价 | presupuesto |
| 样品 | 要样品 | muestra |
| 订单 | 下订单 | pedido, orden |
| 物流 | 问物流 | envío, entrega |
| 付款 | 问付款 | pago, transferencia |
| 投诉 | 投诉问题 | queja, problema |

### 5.4 提醒文案示例

```
📋 待办提醒 (2026-07-02 09:00)
==================================================
🔴 超期待办（2 项）:
  ⚠ [Carlos Mexico] 回复关于交期的疑问
     (已过 4 天)

🔥 高优先级（1 项）:
  ★ [María Argentina] 准备形式发票 截止:2026-07-03

📝 待办（5 项）:
  · [Pedro Spain] 发送样品追踪号
  · [Ana Colombia] 确认 LC 条款

--------------------------------------------------
👥 按客户汇总:
  Carlos Mexico: 3 项待办
  María Argentina: 2 项待办
```

## 6. 常见问题

### Q1: 启动监听提示"未找到微信进程"

**原因**：微信未运行，或进程名不匹配。

**解决**：
- 确保 PC 微信已登录运行
- 在「配置」页检查「微信进程名」：
  - Windows: `WeChat.exe`
  - macOS: `微信`
- Windows 需以**管理员权限**运行软件（右键 → 以管理员身份运行）

### Q2: 提示"未在内存中找到微信密钥"

**原因**：微信刚启动数据库未加载，或版本为 4.1.x。

**解决**：
1. 在微信中**打开几个聊天窗口**（触发数据库加载），再试
2. 如果是微信 4.1.x，内存扫描已失效，需降级到 4.0.x
3. Windows 确保以管理员权限运行

### Q3: 语音转写失败，提示"未找到 SILK 解码后端"

**原因**：缺少 SILK 语音解码库。

**解决**（按系统选择）：
- **Windows**：`pip install pilk`（自带 x64 预编译库）
- **macOS / Linux**：`pip install pysilk-mod`（提供 arm64 / x86_64 wheel）

`silk_decoder.py` 会自动选择可用后端（pilk 优先，回退 pysilk）。打包后通常不会出现此问题。

### Q4: MLX Whisper 在 Windows 上不可用

**正常**：MLX 仅支持 Apple Silicon (M1/M2/M3)。Windows 用户请选择 `volcengine` 引擎。

### Q5: 转写准确率不够高

**优化方法**：
1. 在 config.yaml 的 `initial_prompt` 中加入你的产品名/行业术语
2. 升级到 `large-v3` 模型（更准但更慢）
3. 切换到 `openai` 引擎（最高准确率，西语 WER 2.7%）

### Q6: DeepSeek 分析结果为空

**检查**：
1. 确认 API Key 有效（platform.deepseek.com 查看余额）
2. 确认充值了（新账号需充值才能用）
3. 查看日志是否有 API 调用错误

### Q7: macOS 首次运行提示"无法验证开发者"

**解决**：右键点击 `外贸助手.app` → 选择「打开」→ 确认。或系统偏好设置 → 安全性与隐私 → 允许打开。

### Q8: 监听时微信卡顿

**解决**：在 config.yaml 增大 `poll_interval_ms`（如 200），减少轮询频率。

### Q9: Mac 报"需安装 pymem"

**原因**：pymem 是 Windows 专用库，Mac 上不需要。

**解决**：已在新版本中修复（按平台分支）。Mac 自动用 lldb 提取密钥或引导手动输入。请更新到最新版本。

### Q10: 找不到微信存储目录

**解决**：应用启动时会**自动检测**微信目录，无需手动查找。如未检测到，点击「设置」→「自动检测」按钮。

## 七、仿微信界面使用指南（v2.0+）

新版采用**仿微信三栏界面**，零学习成本。

### 7.1 界面布局

```
┌──────────────────────────────────────────────────────┐
│  外贸助手    微信AI客户管理        [设置] [待办] [MCP] │
├──────────┬───────────────────────┬───────────────────┤
│ 联系人列表│   聊天内容（气泡）     │   AI分析结果      │
│          │                       │                   │
│ 👤 张三   │  ┌──────────┐        │  客户需求分析      │
│ 👤 李四   │  │ 报价$500  │        │  ──────────       │
│ 👥 群聊A  │  └──────────┘        │  摘要: ...         │
│          │       ┌──────────┐   │  需求: 3项         │
│ [搜索]    │       │ 收到,谢谢 │   │  待办: 2项         │
│          │       └──────────┘   │  [保存][导出]      │
├──────────┴───────────────────────┴───────────────────┤
│ 微信已检测: wxid_xxx | ASR: volcengine                │
└──────────────────────────────────────────────────────┘
```

### 7.2 操作流程

1. **自动检测**：启动时自动检测微信目录和进程（无需手动配置）
2. **选择联系人**：左栏点击联系人/群聊，中栏自动加载聊天记录
3. **时间筛选**：选择「今天/近3天/近7天/自定义」过滤消息
4. **语音自动转写**：加载聊天时自动转写所有语音消息
5. **AI分析**：点击「AI分析」按钮，自动转写+DeepSeek分析+生成待办
6. **查看结果**：右栏展示分析结果（摘要/需求/待办/已办）
7. **导出**：点击「导出」保存为 Markdown/JSON/文本

### 7.3 MCP 协议（高级功能）

通过 MCP 协议，可在 Claude Desktop / Trae 中用**自然语言**查询聊天记录。

**配置方法**：
1. 编辑 `~/Library/Application Support/Claude/claude_desktop_config.json`
2. 添加：
```json
{
  "mcpServers": {
    "trade-tools": {
      "command": "/path/to/外贸助手.app/Contents/MacOS/TradeTools",
      "args": ["--mcp"]
    }
  }
}
```
3. 重启 Claude Desktop

**可用自然语言查询示例**：
- "帮我找上周和张三聊的关于报价的记录"
- "列出最近活跃的客户"
- "分析李四的客户需求"
- "转写昨天收到的语音消息"

## 8. 法律与安全提示

### 8.1 使用范围

- 本软件仅可用于解析**本人**或**已获书面授权的员工**的微信数据
- 外贸公司部署时需在员工入职协议中明确告知并取得书面授权

### 8.2 微信封号风险

- 微信数据解析存在一定封号风险（特别是 4.1.x 的 DLL 注入方案）
- 建议：
  1. 锁定微信 4.0.x 版本，关闭自动更新
  2. 用工作号测试，确认无风险后再用于主号
  3. 不要同时运行多个解析工具

### 8.3 数据安全

- 所有数据存储在本地（SQLite），不上传服务器
- 云端 ASR（火山/OpenAI）会传输语音文件到云端，介意隐私请用本地 MLX Whisper
- DeepSeek 仅传输文字（不含语音原文）

### 8.4 法律合规

- 微信数据解析工具存在被腾讯 DMCA/律师函下架的风险
- 本软件仅供内部使用，**不要公开传播或商业化销售**
- 用户需自行承担使用风险
